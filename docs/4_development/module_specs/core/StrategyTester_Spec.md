## 3. 인터페이스 명세
### 3.1 입력
* 입력 유형: 테스트 케이스 정의
* 형식 및 구조: `{strategy_id:str, params:dict, data:DataFrame}`
* 제약 조건: 데이터가 충분해야 함
* 예시: `{"strategy_id":"trend","params":{"ma":10},"data":df}`

### 3.2 출력
* 출력 유형: 테스트 결과 객체

* 예시: `{"pnl":500,"metrics":{"sharpe":1.2}}`
* 반환 조건: 테스트 완료 시

## 4. 내부 처리 로직
1. 입력 데이터와 파라미터로 SimulatorExecutor 호출
2. 수행 결과를 수집하여 성능 지표 계산
